SANDBOX TOOLS REFERENCE
========================
All tools pre-installed in the vr-sandbox image. `cat /tools.txt` to view
this file from inside a running container. Consumer applications (the
ctf-agent runtime, an interactive shell, etc.) may inject additional
domain-specific orientation files alongside this one — read those too if
present.

`uname -m` inside this sandbox reports the SANDBOX's arch, which is the
docker host's arch (often aarch64 on macOS, x86_64 on Linux). It tells
you nothing about a downstream target's arch — always check the actual
binary with `file <path>`.

== NETWORKING ==
nc (netcat)        TCP/UDP connections. nc host port <<'EOF' for heredoc sessions.
curl               HTTP client. -s silent, -k skip TLS, -d data, -H header.
wget               HTTP download. wget -O file url.
nmap               Port scanner. nmap -sV host, nmap -p- host.
masscan            Wide-net port sweeps (faster than nmap for /16+).
                   masscan 10.0.0.0/8 -p1-65535 --rate 10000.
ssh / sshpass      OpenSSH client + non-interactive password auth.
                   sshpass -p guest ssh -o StrictHostKeyChecking=no user@host -p2222.
dig / nslookup     DNS lookups (dnsutils).
whois              WHOIS lookups.
dnsrecon           DNS recon: zone transfers, brute-force, reverse.
                   dnsrecon -d example.com -t std,axfr.
onesixtyone        SNMP scanner. onesixtyone -c community.txt 10.0.0.1
snmpwalk           SNMP walker. snmpwalk -v2c -c public host.
proxychains4       Pivot through socks/http chain.
                   proxychains4 -f /etc/proxychains4.conf nmap target.
chisel-pivot       HTTP-tunneled pivot (jpillora/chisel). On attacker:
                   chisel-pivot server -p 8080 --reverse.
                   On target: chisel-pivot client attacker:8080 R:1080:socks.
mosquitto-clients  MQTT pub/sub. mosquitto_sub -h host -t '#' -v.
socat              Generic stream forwarder. TLS-SNI tunnel:
                   socat TCP-LISTEN:3000,fork,reuseaddr \
                         OPENSSL:host:443,sni=host,verify=0
                   then `nc localhost 3000` to reach the wrapped TCP target.
sc (snicat)        TLS-SNI tunnel helper. sc -b 3000 <sni-host> binds
                   127.0.0.1:3000 and proxies to the SNI'd backend. Use
                   for any TCP target reachable only behind a CNAME-routed
                   SNI front (originally added for bts.wh.edu.pl-style
                   TLS-wrapped TCP challenges).

== BINARY ANALYSIS ==
file               Identify file type. file binary
strings            Extract printable strings. strings -n 8 binary
xxd                Hex dump. xxd file | head, xxd -r to reverse.
hexdump            Hex dump. hexdump -C file.
binwalk            Firmware/embedded extraction. binwalk -e file to extract.
objdump            Disassembly. objdump -d binary (from binutils-multiarch
                   so all common targets are supported — `-m i386:x86-64`,
                   `-m arm`, `-m mips`, etc.). Required for one_gadget on
                   amd64 from an arm64 host sandbox.
llvm-objdump       LLVM-based disassembler. Better at exotic archs (RISC-V, ARM
                   variants, Mach-O). llvm-objdump -d binary,
                   llvm-objdump --triple=aarch64-linux-gnu -d firmware.
ld.lld             LLVM linker. Used for kernel-pwn payloads, custom linker
                   scripts, freestanding builds. `ld.lld -T script.ld -o out a.o`.
cpio               Initramfs unpack/repack for kernel-pwn:
                     zcat initramfs.cpio.gz | cpio -idmv          # extract
                     find . | cpio -o -H newc | gzip > new.cpio.gz # repack
zig                Zig toolchain at /opt/zig (binary tarball, version 0.14).
                   `zig cc -target x86_64-linux-musl -static -o exploit exploit.c`
                   for tiny static binaries; `zig build-exe` for native zig.
                   Useful for kernel exploits, no-libc shellcode loaders.
gh                 GitHub CLI. `gh auth login`, `gh pr create`, `gh workflow run`,
                   `gh api repos/owner/repo/...`. Required for Supply Chain /
                   GitHub Actions challenges where the model needs to push a
                   PR or trigger a workflow on an attacker fork.
ndisasm            Netwide disassembler from NASM. Bare-byte x86 disasm — no
                   ELF parsing required. ndisasm -b 64 shellcode.bin.
nasm               Netwide assembler. nasm -f elf64 shell.asm -o shell.o.
readelf            ELF header info. readelf -a binary.
nm                 Symbol listing. nm binary.
upx                UPX unpacker. upx -d packed.exe (works on ELF + PE).
diec / die         Detect-It-Easy: packer / compiler / protector ID.
                   diec binary  (CLI; falls back to GUI invocation if missing).
ghidra-headless    Batch RE via Ghidra's analyzeHeadless. Standard recipe:
                     mkdir /tmp/proj && \
                     ghidra-headless /tmp/proj P -import binary \
                       -postScript ListSymbols.java -overwrite
                   Also available as `ghidra` and `analyzeHeadless`.
                   Full Ghidra at /opt/ghidra; GHIDRA_INSTALL_DIR is set so
                   `pyghidra.start()` boots without configuration. From
                   Python:
                     import pyghidra
                     pyghidra.start()
                     from ghidra.app.decompiler import DecompInterface
                     with pyghidra.open_program('binary') as api:
                         d = DecompInterface(); d.openProgram(api.currentProgram)
                         res = d.decompileFunction(fn, 60, None)
                         print(res.getDecompiledFunction().getC())
redress            Go binary metadata + symbol/type recovery (uses goretk/gore lib).
                   redress info binary  /  redress source binary
patchelf           Modify ELF interpreter / RPATH. patchelf --set-interpreter ./ld.so bin
yara               Pattern matcher (CLI). yara rules.yar target/ -r
                   Ruleset preloaded at /opt/yara-rules/.

== DECOMPILERS ==
ghidra / ghidra-headless
                   See BINARY ANALYSIS. analyzeHeadless ... -postScript Decompile.java.
                   Also via pyghidra (GHIDRA_INSTALL_DIR is set).
r2 + r2ghidra +    radare2 with two decompiler plugins loaded.
r2dec               - r2ghidra (Ghidra-quality C):
                       r2 ./binary
                       [0x...]> aaa            (analyse all)
                       [0x...]> pdg @ main     (decompile via Ghidra)
                     - r2dec (JS-based pseudo-C):
                       [0x...]> pdd @ main     (decompile via r2dec)
                       [0x...]> pdc @ main     (terser pseudo-C)
                   Try `pdg` first; if output is empty/garbled, switch
                   to `pdd`. One-shot: r2 -qc 'aaa; pdg @ main' ./binary
r2pipe (python)    Drive r2 from Python — useful for batch-decompiling
                   many functions or post-processing output:
                     import r2pipe; r = r2pipe.open('./binary'); r.cmd('aaa');
                     out = r.cmd('pdg @ main')
retdec             Open-source LLVM-based decompiler (amd64 sandboxes only).
                     retdec-decompiler binary -o out.c
                   On arm64 sandboxes use ghidra-headless or r2 pdg instead.
rust-decompile     Rust-specific decompiler (Oxidizer, in angr >=9.2.217).
                   FAR cleaner than Ghidra/r2 on Rust binaries: recovers enums,
                   match/if-let, Option/Result, ?-error-propagation, and macros
                   (println!/format!/panic!). Look for rustc/cargo strings first,
                   then:
                     rust-decompile ./binary              (auto-picks functions)
                     rust-decompile ./binary --functions main 0x4012a0
                   Use this FIRST on Rust binaries; type DBs cover rustc 1.39-1.93.
                   (Runs in an isolated py3.12 venv; the main-python `import angr`
                   stays the C-style symbolic-exec stack.)
ilspycmd           .NET decompiler. ilspycmd app.dll -o out/   (or -lv 8.0 etc.)
jadx               Java/APK decompiler (primary). jadx -d out/ app.jar / app.apk
                   GUI variant available as jadx-gui (X required).
cfr                Alternative Java decompiler — try when jadx mangles
                   control flow on obfuscated bytecode.
                   cfr foo.jar > out.java   (or cfr foo.class)
fernflower         JetBrains' decompiler — strong on lambdas / streams /
                   modern Java. fernflower input.jar /outdir/
                   Output is .java files in <outdir>/<input-basename>/.
                   (Wraps /opt/jdk-21 since fesh0r's tip needs JDK 21+.)
procyon            Fourth-fallback Java decompiler — sometimes handles
                   edge cases (lambdas, switch-on-string) jadx + cfr +
                   fernflower all mishandle. procyon foo.jar > out.java
javap              JVM bytecode disassembler. javap -c -p ClassName, javap -v -p Class.
                   (System default JDK is 11; alternates at /opt/jdk-17,
                   /opt/jdk-21 for tools/code that need newer language
                   features. Run as `/opt/jdk-21/bin/javap ...` etc.)
uncompyle6         Python <=3.8 .pyc decompiler. uncompyle6 file.pyc
decompyle3         Python 3.7-3.8 .pyc decompiler (alternative).
pycdc              Python .pyc decompiler that handles 3.10+. pycdc file.pyc
                   Disassembly: pycdas file.pyc
pyinstxtractor-ng  Extract a PyInstaller-packed exe back to .pyc + assets.
                   python3 -m pyinstxtractor app.exe   (output in app.exe_extracted/)

== CROSS-ARCH / WINDOWS BINARIES ==
qemu-*-static      Run foreign-arch Linux ELFs directly. One static binary per
                   guest arch — pick by name. Examples:
                     qemu-aarch64-static ./binary   (ARM64 ELF on amd64 host)
                     qemu-x86_64-static  ./binary   (x86_64 ELF on arm64 host)
                     qemu-mips64-static  ./binary   (MIPS64, e.g. router fw)
                     qemu-arm-static     ./binary   (32-bit ARM, e.g. IoT)
                     qemu-riscv64-static ./binary   (RISC-V)
                     qemu-i386-static    ./binary   (32-bit x86)
                   The `-static` suffix is optional — `qemu-i386 ./binary`,
                   `qemu-arm ./binary`, etc. are aliased to the static
                   variants for ergonomic invocation.
                   i386 sysroot: /opt/i386-sysroot is populated with
                   libc6:i386 + libstdc++6:i386 etc., and QEMU_LD_PREFIX
                   points there. Dynamically-linked 32-bit x86 ELFs (most
                   pwnable.tw / picoCTF / wargame binaries) Just Work via
                   `qemu-i386-static ./binary` with no -L flag needed. The
                   challenge-shipped libc.so.6 (when present in distfiles)
                   takes precedence; for ROP / one_gadget, use that one.
                   x86_64 sysroot: /opt/x86_64-sysroot mirrors the same
                   pattern for 64-bit x86 ELFs on the arm64 sandbox.
                   /lib64/ld-linux-x86-64.so.2 is symlinked into place,
                   so `./binary` (binfmt) and `qemu-x86_64-static ./binary`
                   both resolve their interpreter without -L flags.
                   Cross-arch GDB: `qemu-x86_64-static -g 1234 ./binary`,
                   then in another shell: `gdb-multiarch ./binary`,
                   `target remote :1234`. Strace works the same way.
                   binfmt_misc may also auto-dispatch ./binary directly when
                   the host kernel has handlers registered (Docker Desktop
                   normally does); explicit qemu-*-static is the fallback.
                   Multi-version GLIBC: when a binary requires a libc the
                   default sysroot doesn't have (`version GLIBC_2.X not
                   found`), use one of the pre-extracted versions under
                   /opt/glibc/<ver>/ (2.23, 2.27, 2.31, 2.35, 2.39).
                   Convenience wrapper:
                     glibc-run                     # list versions
                     glibc-run 2.27 ./vuln         # run with glibc 2.27
                     glibc-run 2.39 ./vuln arg1
                   Match the version to the binary's stated requirement
                   (`strings binary | grep GLIBC_` or check the libc
                   shipped alongside it). Per-challenge libcs from
                   distfiles still take precedence — only reach for
                   /opt/glibc/ when the challenge didn't bundle one.
wine / wine64      Run Windows PE binaries. wine64 file.exe (x86_64 PE),
                   wine file.exe (32-bit PE; available on amd64 sandboxes).
                   Pre-initialised Wine prefix at /opt/wineprefix.
                   Headless: WINEDEBUG=-all wine64 file.exe (already default).
                   On an arm64 sandbox, wine handles ARM PEs natively; x64
                   PE execution under Wine on arm64 needs box64/FEX (not
                   installed) — for those, prefer static analysis (Ghidra,
                   capa, floss) or rebuild the sandbox with --platform
                   linux/amd64.

== CALLBACK / TUNNELS ==
For SSRF, blind XSS, OAuth callbacks, reverse shells, OOB exfil, etc.
Two scenarios:

  1. Target is on our VPN or shares a docker network -> bind a listener
     on tun0/eth0 directly; use the interface IP as the callback URL.
     `expose-port <port>` prints the right address.

  2. Target is on the public internet -> tunnel the local listener out.
     None of these need an account or API key.

expose-port        Helper. `expose-port <port>` prints reachable VPN/docker
                   addresses for <port> plus copy-paste tunnel commands.
                   Does not spawn anything itself.
cloudflared        HTTP(S) tunnel to *.trycloudflare.com, no auth.
                   `cloudflared tunnel --url http://localhost:8080 --no-autoupdate`
                   public URL appears on stderr. Ephemeral; dies with process.
                   Full request/response — serve any payload from the local server.
bore               Raw TCP forward to bore.pub:RANDOM, no auth.
                   `bore local 4444 --to bore.pub`
                   Use for reverse shells, custom protocol servers, anything TCP.
                   remote_port is logged on stderr.
pinggy             SSH-based tunnel; no install — uses the system ssh.
                   60-min sessions on the free tier (anonymous).
                   HTTP/HTTPS:
                     `ssh -p 443 -o StrictHostKeyChecking=no -R0:localhost:8080 a.pinggy.io`
                   TCP (use for reverse shells, custom protocol servers):
                     `ssh -p 443 -o StrictHostKeyChecking=no -R0:localhost:4444 tcp@a.pinggy.io`
                   The assigned public URL (https://<rand>.free.pinggy.link
                   or tcp://<host>:<port>) is printed on the ssh banner —
                   tee stdout/stderr to a file and grep for `.pinggy.`.
                   Useful when cloudflared is rate-limited or bore.pub
                   is blocked; survives where one of the others doesn't.
interactsh-client  Out-of-band capture for DNS / HTTP / SMTP / LDAP, no auth.
                   `interactsh-client -v` prints a *.oast.* hostname; any hit
                   on it is logged. Capture-only — cannot serve dynamic
                   responses (use cloudflared for that). Public OAST servers
                   run by Project Discovery (oast.pro/site/live/...).

  Windows PEs via wine/wine64 — see the section above for invocation.

  starting point for non-trivial RE; dynamic analysis is a complement.

  consumers (ctf-agent, interactive shells, etc.) typically bind a
